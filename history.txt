Baseline model
--------------
- Generator:
  - z_dim: 512
  - native output: 256x256 RGB
  - ResNet upsampling stack: 4 -> 8 -> 16 -> 32 -> 64 -> 128 -> 256
  - channels: 512, 512, 512, 512, 256, 128, 64
  - GroupNorm
  - self-attention at 32x32
- Discriminator:
  - ResNet downsampling stack from 256 to 4
  - SpectralNorm
  - MinibatchStd
  - self-attention at 32x32
- Loss:
  - non-saturating logistic GAN loss
  - R1 regularization
- EMA generator is used for sampling.

512 freeze experiment
---------------------
- Previous 512 tests from the 256 fine-tuned checkpoint produced worse samples:
  color bleeding, texture artifacts, and distorted faces.
- A second 512 test from the original baseline checkpoint still showed weak
  learning signals:
  - D outputs stayed near 0
  - G/D losses stayed close to log(2)
  - gradient norms were very small
- Added freeze_g_backbone support in train.py.
- Added configs/finetune_512_freeze_colab.yaml.
- Freeze strategy:
  - load original ffhq256_baseline.pt
  - freeze Generator input projection and 4->256 blocks
  - train only the 512 Generator block plus out_norm/to_rgb
  - train the full Discriminator
- Initial freeze config:
  - lr_g: 0.0002
  - lr_d: 0.0005
  - r1_lazy_every: 128
  - ckpt_every: 10_000
  - total_images: 300_000

512 wide freeze experiment
--------------------------
- The narrow freeze run kept face structure better than full-G training, but
  quality degraded after 10k images:
  - grid_000010000.png looked best
  - later checkpoints increased green/purple tint and stripe artifacts
- Hypothesis: the 512 block with 64 channels is too small, so it learns
  color/texture artifacts instead of useful high-resolution detail.
- Added configs/finetune_512_wide_freeze_colab.yaml.
- Changes from the narrow freeze config:
  - Generator 512 channels: 64 -> 128
  - Discriminator 512 channels: 64 -> 128
  - final_channels: 64
  - freeze_g_backbone: true
  - trainable_g_resolutions: [512]
  - lr_g: 0.0001
  - lr_d: 0.0005
- The first wide attempt without final_channels failed because changing
  512 channels from 64 to 128 also changed the RGB head shape. That forced
  to_rgb to random initialization and caused severe color collapse.
- Added Generator final_adapter support:
  - high-resolution block can output 128 channels
  - 1x1 final_adapter maps 128 -> 64
  - baseline out_norm and to_rgb can still be partially loaded
- Smoke test command:
  python train.py \
    --config configs/finetune_512_wide_freeze_colab.yaml \
    --init-from /content/drive/MyDrive/project2/ckpt/ffhq256_baseline.pt \
    --total-images 50000
- Result: failed. The first wide run without final_channels caused severe
  green/purple color collapse because to_rgb was randomly initialized.
- A second wide run with final_adapter and final_channels=64 still failed:
  the random 128->64 adapter changed the feature distribution expected by the
  pretrained RGB head, producing even stronger color artifacts.
- Decision: abandon wide 512 for now.
- Removed the failed wide config and reverted the final_adapter/final_channels
  code path to keep the main model simple.
- Return to narrow 512 channels:
  - Generator 512 channels: 64
  - trainable_g_resolutions: [512]
  - regenerate ckpt_000010000.pt from baseline
  - then try partial unfreeze [256, 512] with low G lr

512 residual progressive experiment
-----------------------------------
- Narrow freeze and partial-unfreeze runs still accumulated color tint and
  stripe artifacts. The main issue is that a newly initialized high-resolution
  block immediately replaces the RGB output path.
- Added an optional residual progressive RGB path in src/model.py.
- The original baseline RGB head remains the base image path:
  - render the pretrained 256 RGB image
  - bilinear upsample it to 512
  - add a small learned 512 residual refinement
- The new residual RGB convolution is zero initialized, so the initial output
  matches the upsampled baseline instead of replacing it with random colors.
- Added configs/finetune_512_residual_colab.yaml:
  - Generator 512 channels: 128
  - residual_rgb_resolutions: [512]
  - residual_rgb_scale: 0.1
  - freeze_g_backbone: true
  - trainable_g_resolutions: [512]
  - lr_g: 0.0001
  - lr_d: 0.0003
  - ckpt_every: 10_000
- Smoke test command:
